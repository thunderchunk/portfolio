Roster Rooster Web App

This web app was a student project. The app allows for an employer to schedule a team of people with different time requirements and post those schedules to users. There are two layers of login: admin and user. The admin controls the scheduling for a company, and the users can see when they are scheduled and request time off.

Unhappy with the calendar plugins I found for Node, and the ready-made templates for AngularJS on Github too unfriendly for mobile, I made my own. Using the basic Javascript time object, I built a calendar that would dynamically generate a grid with the corresponding days on the correct column.

I then built a database in SQL with five tables, each corresponding to an employer and the employee's time requirements. Each employee can enter their available hours, a weekly minimum and maximum, and stop and start times for each day of the week.

The admin can then go into their interface, edit employee data, schedule a new day by populating the schedule with a bar representing an employee's available work time, edit the bar to fit their needs, save, and publish the day.

I spent far less time working on the front end of this project and almost all the time on the logic. This app is almost 60% raw Javascript. I had to learn the Javascript time object inside and out, as well as all of its strange and archaic parsing errors when talking to AngularJs and SQL, and how to generate pictures from numbers.

Initially, I studied the D3 library, learning how to manipulate data using their standards only to find it insufficient for the task at hand. If you venture too far outside of a bar graph, a pie chart, or a relationship graph, you need a working knowledge of calculus, which I lack.

It only works on Android and Chrome at the moment, and I plan on finishing the app to a production ready product on Ionic. I'm making it mobile to accommodate small business owners with large part-time employee bases. I disabled the login and username so anyone who's interested in playing with Roster-Rooster can if they want to. It's all hooked up to a database, so enjoy.

I used AngularJS, Node, JQuery, Javascript, SCSS, HTML, Gulp, Ui-Router, and SQL
